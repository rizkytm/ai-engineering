# Exercise 01: Portfolio, Proposals & Pricing

**Time:** 3–4 hours
**Goal:** Create real portfolio assets and proposal documents you can use immediately.

---

## Part 1: Create a GitHub README for an AI Project

**Pick one project from your portfolio (Modules 1–19). Create a professional README.**

### Requirements

1. Create a new file `README.md` in your project directory
2. Include all sections from the template in Notebook 01:
   - One-line value description
   - Problem statement (business context)
   - Solution overview
   - Architecture diagram (ASCII art or image)
   - Key features with business value
   - Demo (link or screenshots)
   - Getting started instructions
   - Results with metrics
   - What you learned
   - Tech stack table
   - Contact information

### Checklist

- [ ] No hardcoded API keys
- [ ] `.env.example` file exists
- [ ] Demo link works (or screenshots included)
- [ ] Metrics are specific (numbers, not adjectives)
- [ ] Problem is stated in business terms, not technical
- [ ] Getting started instructions work from scratch
- [ ] Contact info is included

### Grading

| Criteria | Points |
|----------|--------|
| Problem statement clarity | /10 |
| Solution description | /10 |
| Documentation completeness | /10 |
| Demo/screenshots quality | /10 |
| Professional presentation | /10 |
| **Total** | **/50** |

## Part 2: Write a Portfolio Project Description

**Write 3 different descriptions for the same project:**

1. **Technical audience** (100 words) — focus on architecture, tools, approach
2. **Business audience** (100 words) — focus on problem, results, ROI
3. **Elevator pitch** (25 words) — one sentence that hooks

### Example

**Project:** RAG Customer Support System

**Technical:** "Built a RAG pipeline using LangChain with hybrid search (BM25 + semantic embeddings). ChromaDB for vector storage, OpenAI for generation. Deployed on AWS Lambda with API Gateway. Achieved 92% retrieval accuracy on benchmark dataset."

**Business:** "Reduced customer support response time from 8 minutes to 2 minutes by building an AI assistant that searches 10,000+ internal documents instantly. Saved an estimated $50K/year in support costs."

**Elevator pitch:** "I built an AI that lets support agents find answers in 2 seconds instead of 15 minutes."

### Submit

Write your three descriptions and save them to a file in this directory.

## Part 3: Draft a Proposal for a Sample AI Project

**Scenario:**

> A mid-size e-commerce company (500 employees) wants to build an AI-powered product recommendation system. They have:
> - 100,000 products with descriptions
> - 2 years of purchase history (500K transactions)
> - Customer browsing data
>
> Goal: Increase average order value by 10% through personalized recommendations.

### Requirements

Using the proposal template from Notebook 02, create a complete proposal including:

1. **Understanding of their challenge** (show you listened)
2. **Proposed solution** (high-level, not implementation details)
3. **Scope & deliverables** (in scope, out of scope, revision limits)
4. **Timeline & milestones** (realistic 6-8 week timeline)
5. **Pricing** (provide both fixed and hourly options)
6. **Why you** (relevant experience)

### Save

Save your proposal as `proposal-sample.md` in this directory.

## Part 4: Create a Pricing Estimate

**Using the pricing calculator from Notebook 02, estimate the project above.**

### Assumptions to Use

- Your hourly rate: $150/hr
- Estimated hours for each phase:
  - Discovery & data audit: 20 hours
  - Model development: 40 hours
  - API & integration: 30 hours
  - Testing & optimization: 20 hours
  - Documentation & handover: 10 hours
- Complexity multiplier: 1.3
- Buffer: 25%
- Profit margin: 20%

### Calculate

1. Base cost (sum of hours × rate)
2. Complexity-adjusted cost
3. Cost with buffer
4. Final price with margin
5. Effective hourly rate

### Document

Write out your calculation and explain your pricing decisions. Save as `pricing-estimate.md`.

## Bonus: Set Up a Complete Portfolio on GitHub

**Time:** 2–3 additional hours

### Steps

1. **Create a profile README repo**
   - Repo name must match your GitHub username
   - Add a professional README with your focus areas

2. **Pin 6 repos**
   - Best RAG project
   - Best agentic AI project
   - Best end-to-end pipeline
   - Best domain-specific project
   - Open source contribution
   - Most popular project (by stars)

3. **Polish each pinned repo**
   - Working demo link
   - Professional README
   - Clean commit history

4. **Set up a simple portfolio site**
   - One page with projects, about, contact
   - Deploy on GitHub Pages

### Verification

- [ ] Profile README exists and displays on your profile
- [ ] 6 repos are pinned with good descriptions
- [ ] Each pinned repo has a working README
- [ ] Portfolio site is live at a public URL
- [ ] All demo links work
