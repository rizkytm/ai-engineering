| **ESLint** | JavaScript | Linting | https://eslint.org |
| **Prettier** | JS/TS | Code formatting | https://prettier.io |

### Code Quality Metrics

- **Cyclomatic Complexity** — Measure code complexity (target < 10 per function)
- **Code Coverage** — Target > 80% for critical paths
- **Technical Debt Ratio** — SonarQube measurement of maintainability
- **Cognitive Complexity** — How hard code is to understand (SonarQube)

### Measuring AI Impact

Track these metrics to measure how AI affects your coding:

- **Time to first working version** — How fast can you get a prototype?
- **Bug rate** — Bugs per 1000 lines (AI vs manual)
- **Code review cycles** — Fewer iterations = better AI output
- **Test coverage** — AI can help increase coverage faster
- **Refactoring speed** — Time to improve existing code

### Quick Prompts

**Debug**: "Here's the error [trace]. Here's the code [code]. What's wrong?"
**Refactor**: "Refactor this to [goal]. Keep the same behavior."
**Explain**: "Explain this code line by line for a junior developer."
**Test**: "Write tests covering happy path, edge cases, and errors."
**Review**: "Review for security, performance, and correctness."
